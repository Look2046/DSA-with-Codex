### 13. 交互式可视化指南

```markdown
## 交互式可视化指南

### 可视化特性

本模块提供以下可视化功能：

- **动画展示**：自动演示算法执行过程
- **逐步调试**：单步执行，观察每一步的变化
- **代码同步**：代码和动画自动同步
- **统计信息**：实时显示比较次数、交换次数等

### 如何使用可视化

#### 1️⃣ 准备数据

**随机生成**：
- 点击"生成随机数据"按钮
- 选择数据大小：小 (10) / 中 (50) / 大 (100)
- 点击"确认"

**自定义输入**：
- 点击"手动输入"
- 输入逗号分隔的数字：`3,1,4,1,5,9,2,6`
- 点击"确认"

**预设数据**：
- 已排序：`1,2,3,4,5`
- 逆序：`5,4,3,2,1`
- 随机：`3,1,4,1,5`

#### 2️⃣ 开始动画

**自动播放**：
- 点击"播放"按钮
- 观看算法自动执行
- 可调整播放速度：0.5x, 1x, 1.5x, 2x

**逐步执行**：
- 点击"下一步"按钮
- 每次执行一个步骤
- 观察代码面板中的高亮变化

#### 3️⃣ 观察细节

**节点颜色含义**：
- 🟡 黄色（Comparing）：正在比较的元素
- 🟢 绿色（Moving）：正在交换/移动的元素
- 🔵 蓝色（Visiting）：当前访问的位置
- 🟣 紫色（Matched）：已完成的元素
- ⚪ 灰色（Idle）：未处理的元素

**统计面板**：
- 比较次数：当前已进行的比较操作数
- 交换次数：当前已进行的交换操作数
- 总操作数：所有操作的总数

**代码高亮**：
- 绿色背景：即将执行的代码行
- 蓝色背景：刚刚执行过的代码行

#### 4️⃣ 暂停与继续

- 点击"暂停"可暂停动画
- 点击"继续"恢复播放
- 使用"上一步"返回到之前的状态

#### 5️⃣ 重置与重新开始

- 点击"重置"清除所有操作
- 使用相同的输入数据重新开始
- 或者"生成新数据"并重新开始

### 常见问题

**Q1：为什么动画很快看不清？**  
A：调整播放速度到 0.5x，或使用逐步执行功能。

**Q2：如何查看具体是哪一行代码在执行？**  
A：打开代码面板，动画会实时高亮对应的代码行。

**Q3：能否比较两个不同算法的执行过程？**  
A：可以。打开两个浏览器标签页，分别运行不同的算法，进行对比。

### 最佳实践

1. **第一遍**：在自动播放模式下快速浏览，了解整体流程
2. **第二遍**：使用逐步执行，细致观察每个步骤
3. **第三遍**：对比不同的输入数据（已排序、逆序、随机），观察差异
4. **第四遍**：尝试手动输入特定的数据，验证自己的理解
```

## 编写规范

### 通用规范

1. **标题层级**
   - `#` - 模块标题（仅一个）
   - `##` - 主要章节
   - `###` - 子章节
   - `####` - 详细小节

2. **代码块**
   - 指定语言：`javascript`, `python`, `pseudocode`, `json`, `typescript`
   - 包含详细注释
   - 提供完整的可运行示例

3. **列表和表格**
   - 有序列表用于步骤
   - 无序列表用于特性/要点
   - 表格用于对比信息

4. **特殊标记**
   - ✅ 优点/成功
   - ❌ 缺点/失败
   - ⚠️ 警告/注意
   - 📌 重要标记
   - 💡 提示
   - 🎯 目标

5. **链接**
   - 内部链接：`[链接文本](#锚点)`
   - 外部链接：`[链接文本](http://...)`
   - 模块引用：`[L-03](#)`（指向其他模块）

### 内容质量要求

| 要求 | 标准 |
|-----|------|
| 总字数 | 3000-5000字 |
| 代码示例数 | 至少5个 |
| 插图/图表 | 至少3个 |
| 常见题目 | 至少3个 |
| 外链参考 | 至少2个 |
| 代码语言 | JavaScript + Python |

### 可读性规范

- 段落长度：每段100-300字
- 句子长度：平均15-20字
- 避免过长的句子（>50字）
- 使用短段落和分点符号
- 每个概念用加粗强调一次
